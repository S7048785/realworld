# 文章列表移动的悬浮指示器动画实现

## 原理总结

鼠标悬浮在文章项上时，计算该项的位置和尺寸，动态调整一个半透明背景层的顶部位置、高度和透明度，实现平滑的跟随指示效果。

## 效果描述

在文章列表页面，当用户鼠标悬浮在某个文章项上时，会出现一个半透明的背景指示器。该指示器会平滑地跟随鼠标位置，显示在悬浮文章项的区域内，形成一个视觉上的高亮效果。指示器具有圆角边框和淡入淡出的动画过渡。

## 实现原理

1. 状态初始化：hoverCard 状态初始为 { top: 0, height: 0, opacity: 0 }，指示器不可见。

```tsx
const [hoverCard, setHoverCard] = useState({
  top: 0,
  height: 0,
  opacity: 0,
});
```

2. 鼠标进入事件：当鼠标进入某个 ArticleItem 时，获取当前元素的 offsetHeight 和 offsetTop，设置 hoverCard 的位置和高度，并将 opacity 设置为 0.5。

```tsx
const handleCardMouseEnter = (
  index: number,
  event: React.MouseEvent<HTMLDivElement>,
) => {
  setHoverCard({
    height: event.currentTarget.offsetHeight,
    top: event.currentTarget.offsetTop,
    opacity: 0.5,
  });
};

{
  articleList.map((article, index) => (
    <ArticleItem
      key={article.id}
      article={article}
      onLike={handleLike}
      onMouseEnter={(event) => handleCardMouseEnter(index, event)}
    />
  ));
}
```

3. 动画执行：指示器在鼠标进入时淡入并定位到对应文章项的位置，鼠标离开时淡出。动画使用缓动函数 cubicBezier(0.4, 0, 0.2, 1)，持续时间 0.3 秒，包含刚度和阻尼参数以实现自然过渡。

```tsx
<motion.div
  transition={{
    stiffness: 0.15,
    damping: 0.7,
    ease: cubicBezier(0.4, 0, 0.2, 1),
    duration: 0.3,
  }}
  animate={{
    top: hoverCard.top,
    height: `${hoverCard.height}px`,
    opacity: hoverCard.opacity,
  }}
  className="absolute left-0 w-full bg-sidebar-ring/50 dark:bg-jade-800/20 rounded-default pointer-events-none -z-10"
></motion.div>
```

4. 鼠标离开事件：指示器仅在文章列表容器内响应鼠标事件，当鼠标离开整个列表容器时，将 opacity 设置为 0，指示器淡出。

```tsx
const handleCardMouseLeave = () => {
  setHoverCard((prev) => ({ ...prev, opacity: 0 }));
};
```

5. 渲染逻辑：指示器作为绝对定位元素放置在列表顶部，z-index 为 -10，确保不干扰其他元素交互。
6. 位置计算：使用 event.currentTarget.offsetTop 和 event.currentTarget.offsetHeight 获取相对位置，确保指示器精确对齐。
7. 性能优化：动画仅在必要时触发，避免频繁重渲染。使用 pointer-events-none 防止指示器干扰鼠标事件。

## 渲染优化：引入 mitt 解耦指示器与列表渲染

### 背景与痛点

- 若把 hoverCard 状态放在列表父组件或列表项中，通过 setState 更新会导致父组件重新渲染，从而触发所有 ArticleItem 重渲染，列表在快速移动鼠标时出现不必要的高频重绘。

### 解决思路

- 使用轻量事件总线 mitt 解耦指示器的状态更新与文章列表渲染：
  - ArticleItem 只在鼠标进入时发送事件，不持有指示器状态；
  - ArticleListTabsContent 在鼠标离开整个列表时发送结束事件；
  - ArticleItemIndicator 独立订阅事件，内部维护 hoverCard 状态并驱动动画；
  - 列表和列表项不再因为指示器动画而重渲染。

### 事件约定

- card-hover：载荷为 React.MouseEvent<HTMLDivElement>，用于读取当前项的 offsetTop 与 offsetHeight；
- card-hover-end：无载荷，指示器淡出。

### 代码要点

ArticleItem 在鼠标进入时发送事件：

```tsx
// ArticleItem.tsx
onMouseEnter={(event) => {
  emitter.emit("card-hover", event);
}}
```

文章列表容器在鼠标离开时发送结束事件：

```tsx
// ArticleListTabsContent.tsx
<div onMouseLeave={() => emitter.emit("card-hover-end")}>
  <ArticleItemIndicator />
  {/* ... */}
</div>
```

指示器组件独立订阅事件并维护自身状态：

```tsx
// ArticleItemIndicator.tsx
useEffect(() => {
  const handleCardMouseEnter = (event: React.MouseEvent<HTMLDivElement>) => {
    setHoverCard({
      height: event.currentTarget.offsetHeight,
      top: event.currentTarget.offsetTop,
      opacity: 0.5,
    });
  };
  const handleCardMouseLeave = () => {
    setHoverCard((prev) => ({ ...prev, opacity: 0 }));
  };
  emitter.on("card-hover", (event: unknown) => {
    handleCardMouseEnter(event as React.MouseEvent<HTMLDivElement>);
  });
  emitter.on("card-hover-end", handleCardMouseLeave);
}, []);
```

### 效果与收益

- 重渲染范围缩小：只有 ArticleItemIndicator 跟随动画更新；ArticleItem 与父列表保持静默；
- 更强的解耦：事件驱动跨组件通信，便于后续扩展（如键盘导航、触摸高亮）；
- 动画流畅：在高频悬浮切换时减少状态抖动与渲染压力。

### 实践建议

- 清理监听：若指示器组件存在卸载/重挂载场景，可在 useEffect 返回函数中取消订阅，避免重复注册；
- 定位上下文：列表容器需保证有合适的定位上下文（如相对定位），使绝对定位的指示器正确覆盖列表区域；
- 事件载荷：也可选择发送 { top, height } 纯数据载荷，进一步降低对 DOM 事件对象的依赖；
- 边界处理：快速移入移出列表时，确保在 card-hover-end 到来时能及时将 opacity 置 0。
